# AI Agent Rules

1. **Lint Fixes**: If any source code modification is made, the agent must ensure all lint issues in the modified files are fixed using `eslint --fix` or manual edits.
2. **TypeScript Integrity**: If any source code modification is made, the agent must ensure all TypeScript type issues in the modified files are resolved. Run type-checking (e.g., `tsc --noEmit`) to verify.
3. **Responsive Design**: Ensure all web components follow the project's design aesthetics (premium, dynamic, and responsive).
4. **E2E Verification**: Always allow and prioritize E2E script/browser verification to ensure that new modules and UI changes work as expected in a real browser environment.
5. **Data Management**: Use `TypeOrmDataManager` for all database interactions.
